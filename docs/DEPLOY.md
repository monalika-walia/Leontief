# docs/DEPLOY.md — hosting the landing + dApp (Vercel)

Both front-ends are static and deploy to Vercel on the `29projectslab` team.

## dApp (`app/`)

The dApp bakes the live testnet contract IDs into the bundle at build time
(Vite reads `app/.env.local`, generated by `scripts/gen_app_env.sh` from
`deploy.env`). Deploy the built bundle:

```sh
just app            # or: ./scripts/gen_app_env.sh && cd app && npx vite build
cd app/dist
# SPA fallback so client routes resolve:
echo '{ "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }' > vercel.json
vercel deploy --prod --yes --name leontief-app
```
→ https://leontief-app.vercel.app

## Landing + litepaper (`landing/`)

Deploy the static files **from a clean directory** — Vercel otherwise mis-detects
the `public/` folder as the output root and serves 404s (`.vercelignore` guards
against this for direct deploys):

```sh
vercel deploy --prod --yes --name leontief-landing --cwd landing
```
→ https://leontief-landing.vercel.app · `/Litepaper.dc.html`

`index.html` mirrors `landing.html` so `/` serves the landing; the nav links keep
their explicit `landing.html` / `Litepaper.dc.html` filenames.

## Deployment protection

New Vercel projects default to **SSO protection on `*.vercel.app`**, which 302s
public visitors to a Vercel login. For public SCF review it must be off:

```sh
# via API (needs the team id):
curl -X PATCH -H "Authorization: Bearer $VERCEL_TOKEN" \
  "https://api.vercel.com/v9/projects/<project-id>?teamId=<team-id>" \
  -H 'Content-Type: application/json' -d '{"ssoProtection": null}'
```
or Dashboard → Project → Settings → Deployment Protection → Vercel Authentication → Disabled.

## Known follow-ups

- **Early-access form**: on the deployed landing, `config.js` finds no Codespaces
  host, so it falls back to `localhost:8787` and the form saves to `localStorage`
  only. To persist to the backend, host `services/api` (Vercel serverless or
  Railway + Postgres) and set `window.LEONTIEF.API_BASE` to that origin (and add
  the landing origin to the API's `CORS_ORIGINS`). Tracked under D4 hosting.
- **Env-based redeploys**: for CI-driven deploys, wire the contract IDs as Vercel
  project env vars instead of baking `app/.env.local` locally.

## Secrets

The Vercel token is never committed. Rotate any token pasted into chat/logs after
use (Vercel → Account → Tokens).
